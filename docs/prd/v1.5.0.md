# PRD: ReddiChat v1.5.0 — Search, Export & Interaction Polish

**Status:** Draft
**Date:** 2026-03-28

## Version Goal

Add conversation search and export, give streaming responses visible phase feedback, introduce keyboard shortcuts, and close the remaining error UI and accessibility gaps.

---

## Features

### 1. Conversation Search

- Search input at the top of the conversation list in `frontend/components/layout/ConversationHistory.tsx`
- Filters the thread list client-side by title; for content search, pass a `?q=` query parameter to the conversations fetch endpoint
- `Cmd/Ctrl + K` focuses the search input (see Keyboard Shortcuts feature)

---

### 2. Conversation Export

- "Export as Markdown" button in the chat area header or via a conversation context menu
- Serialises the current thread's messages to a `.md` file and triggers a browser download
- No API call needed — export is pure client-side from the chat store (`useChatStore`)
- Format: each message rendered as `## User` / `## Assistant` headings followed by message content; Reddit sources appended as a `## Sources` section with URLs

---

### 3. Streaming Feedback Phases

Show distinct status labels during a streaming response in `frontend/components/chat/ChatArea.tsx`:

1. "Searching Reddit..." — tool calls in flight
2. "Analyzing sources..." — tool results received, LLM generating
3. "Generating response..." — text streaming

Additional behaviour:
- Display elapsed time after the response completes (e.g., "Responded in 4.2s")
- Show a warning banner if the response takes >15 s without completing (e.g., "This is taking longer than usual...")

Phase state is derived from the streaming event stream already consumed in `ChatArea.tsx`; no new API surface needed.

---

### 4. Keyboard Shortcuts

| Shortcut | Action |
|---|---|
| `Cmd/Ctrl + K` | Focus conversation search |
| `Cmd/Ctrl + N` | Start a new chat |
| `Escape` | Close dropdown / overlay / modal |

Register globally in `frontend/app/(chat)/layout.tsx` using a `useEffect` with `keydown` listener. Guard `Cmd/Ctrl + N` to avoid hijacking browser new-window on non-Mac. `Escape` delegates to whatever overlay is currently open — check open state from `useUIStore`.

---

### 5. Better Error Messages (UI)

- Map API error codes to human-readable messages with an action hint (e.g., "Rate limit reached — try again in X minutes")
- "Retry" button on failed messages with exponential backoff indicator
- Update `frontend/components/chat/ChatArea.tsx`

Error code mapping (minimum set for v1.5.0):

| Error condition | User-facing message |
|---|---|
| 429 rate limit | "Rate limit reached — try again in X minutes" |
| 401 session expired | "Session expired. Please sign in again." |
| 500 / unknown | "Something went wrong. Please try again." |
| Network offline | "No internet connection. Check your network." |

---

### 6. Accessibility Improvements

- Add missing `aria-label` attributes on all interactive elements across sidebar and chat components
- `aria-busy="true"` on streaming message bubbles
- Visible `focus-visible` ring on all buttons and inputs (audit against Tailwind config to confirm `ring` utilities are applied on `:focus-visible`)
- Ensure all icon-only buttons have an accessible name via `aria-label` or `<span className="sr-only">`

Audit scope: `Sidebar.tsx`, `ConversationHistory.tsx`, `MessageList.tsx`, `ChatArea.tsx`, `MessageInput.tsx`.

## Technical Scope

### New files
- None — all changes are within existing components

### Modified files
- `frontend/components/layout/ConversationHistory.tsx` — add search input
- `frontend/components/chat/ChatArea.tsx` — streaming phases, elapsed time, warning banner, error UI with retry
- `frontend/app/(chat)/layout.tsx` — global keyboard shortcut registration (`Cmd/Ctrl + K`, `Cmd/Ctrl + N`, `Escape`)
- `frontend/components/layout/Sidebar.tsx` — accessibility audit: `aria-label` gaps
- `frontend/components/chat/MessageList.tsx` — accessibility audit; `aria-busy` on streaming bubbles; export trigger surface (if placed in message list header)
- `frontend/components/chat/MessageInput.tsx` — accessibility audit
- `frontend/stores/chat-store.ts` — add export helper that serialises thread messages to Markdown string

### New dependencies
- None — all needed utilities (`date-fns`, `zustand`, `react-hot-toast`) are already present

### Database changes
- None

---

## Out of Scope

- Server-side full-text conversation search (client-side title filter only in v1.5.0)
- Export formats other than Markdown (PDF, JSON — deferred)
- Custom keyboard shortcut configuration
- Screen reader testing beyond basic ARIA attribute coverage
- Streaming phase detection beyond what the existing event stream exposes
