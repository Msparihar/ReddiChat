# PRD: ReddiChat v1.4.0 — Profile Dropdown & UX Polish

**Status:** Reviewed
**Date:** 2026-03-28

## Version Goal

Replace the sidebar footer's bare avatar + logout buttons with a rich profile dropdown, add a copy button on AI responses, and make the chat layout fully usable on mobile.

---

## Features

### 1. Profile Dropdown (Priority)

Replace the current sidebar footer user profile block in `frontend/components/layout/Sidebar.tsx` (lines 252–309) — including the standalone logout button and settings icons — with a single avatar trigger that opens a rich dropdown panel. The standalone logout button and settings icons are fully removed from the sidebar; all their functionality moves into the dropdown.

**Install the component:**
```
npx shadcn@latest add https://21st.dev/r/kokonutd/profile-dropdown
```

The installed component lands in `frontend/components/ui/profile-dropdown.tsx`. Wire it into `Sidebar.tsx` replacing the existing user profile `<div>` block (lines 252–309).

> **Fallback:** If the shadcn component URL is unavailable, manually create a profile dropdown using the existing shadcn `DropdownMenu` + `Avatar` components already installed in the project (`@radix-ui/react-dropdown-menu`, `@radix-ui/react-avatar`).

#### Dropdown sections

| Section | Content | Data source |
|---|---|---|
| Profile header | Avatar image (or initials fallback), display name, email | `useSession()` from `@/lib/auth/client` |
| Usage overview | "X / 100 messages used today" progress bar + reset timestamp | `useUsage()` from `@/hooks/use-usage.ts` |
| Current plan | "Free Plan — 100 msgs/day" badge | Hardcoded for v1.4.0; monetization in v2.1.0 |
| Settings | Change display name, theme toggle (light/dark) | Display name: PATCH `/api/user/profile`; theme: `useTheme()` from `@/components/providers/theme-provider` |
| Billing / Contact | "Contact for billing" — `mailto:` link or `/contact` page link | Static link, no self-serve billing yet |
| Sign out | Calls `signOut()` from `@/lib/auth/client` | Existing `handleLogout` logic |

#### Theme toggle migration

Remove the standalone theme toggle buttons from the sidebar header (both expanded and collapsed states). Theme control moves exclusively into the profile dropdown settings section. The `toggleTheme()` call and `useTheme()` hook remain — only their render location changes.

#### Collapsed sidebar behaviour

When the sidebar is collapsed to icon-only width (`isSidebarOpen === false`), show only the avatar button — clicking it should still open the dropdown anchored to the avatar.

#### Z-index layering

- Mobile sidebar overlay: `z-40` (matches existing `Sidebar.tsx` `z-40` class)
- Profile dropdown: `z-50` — must appear above the sidebar overlay
- Modals / dialogs: `z-[60]` — appear above the dropdown

#### New API endpoint: `PATCH /api/user/profile`

- **File:** `frontend/app/api/user/profile/route.ts`
- **Auth:** Call `auth.api.getSession({ headers: request.headers })`. Return `401 { error: 'Unauthorized' }` if no valid session.
- **Request body:** `{ displayName: string }`
- **Validation:** `z.object({ displayName: z.string().trim().min(1).max(50) })` — return `400 { error: 'Invalid input' }` on failure
- **Rate limiting:** `checkRateLimit('profile:${userId}', RATE_LIMITS.reddit)` — return `429` if exceeded
- **Database update:** `db.update(schema.user).set({ name: displayName }).where(eq(schema.user.id, userId))`
- **Sanitization:** React escapes display values on render; Drizzle uses parameterized queries. No additional HTML stripping is needed.
- **Logging:** `logger.info('profile:update', { requestId, userId })` on success; `logger.error('profile:update:failed', { requestId, error })` on failure
- **Response:** `{ user: { id, name, email, image } }`

#### Session revalidation after display name update

After the PATCH succeeds, call `window.location.reload()`. This is acceptable for an infrequent action like a name change and guarantees the session and all derived UI reflect the new name without additional state management. If the PATCH returns `401`, show a toast: `"Session expired. Please sign in again."` and redirect to the login page.

#### Settings sub-panel inside dropdown

- Display name: text input + save button; calls `PATCH /api/user/profile`
- Theme: toggle switch between light/dark; calls `toggleTheme()` — no API needed, preference persisted in localStorage by the existing theme provider

#### Usage display detail

`useUsage()` already polls `/api/usage` every 60 seconds. Show:
- A linear progress bar: `usage.messages / limits.messages`
- Label: `"X of Y messages used today"`
- Sub-label: `"Resets " + formatDistanceToNow(resetAt, { addSuffix: true })` using `date-fns` (already in `package.json` v4.1.0)
- **Loading state:** `<div className="h-2 w-full bg-gray-200 dark:bg-gray-700 rounded animate-pulse" />`
- **Error state:** `<span className="text-xs text-gray-400">Usage unavailable</span>`

## Technical Scope

### New files
- `frontend/components/ui/profile-dropdown.tsx` — installed via shadcn CLI (or created manually as noted above)
- `frontend/app/api/user/profile/route.ts` — PATCH endpoint for display name update

### Modified files
- `frontend/components/layout/Sidebar.tsx` — replace sidebar footer (lines 252–309) with `<ProfileDropdown>`; remove standalone theme toggle and logout button that move into the dropdown
- `frontend/app/(chat)/layout.tsx` — mobile sidebar overlay logic
- `frontend/components/chat/MessageList.tsx` — copy button per assistant message
- `frontend/components/chat/RedditSource.tsx` — mobile vertical stack

### New dependencies
- Shadcn profile-dropdown component (via CLI, no manual npm install)
- `date-fns` v4.1.0 — already present in `package.json`, no install needed

### Database changes
- None — Better Auth already stores `name` and `image` on the user record; the PATCH endpoint updates `name` only
