# PRD: ReddiChat v1.1.0

**Status:** Shipped
**Date:** 2026-03-28

## Version Goal

Add multi-model support so users can switch between Google Gemini and OpenAI models from the chat input.

## Features

- **Model Selector Dropdown** — Animated dropdown in the message input bar to pick AI model per conversation
- **OpenAI Provider** — Added `@ai-sdk/openai` alongside existing Google Gemini provider
- **Multi-Provider Agent** — `streamChatResponse` dynamically selects provider based on chosen model
- **Model Config** — Centralized model definitions in `lib/ai/models.ts`

## Available Models

| Display Name | Model ID | Provider |
|---|---|---|
| Gemini 2.5 Flash (default) | `gemini-2.5-flash` | Google |
| Gemini 3 Flash | `gemini-3-flash-preview` | Google |
| Gemini 3.1 Pro | `gemini-3.1-pro-preview` | Google |
| GPT-5.4 Mini | `gpt-5.4-mini` | OpenAI |
| GPT-5.4 | `gpt-5.4` | OpenAI |

## Technical Scope

- **New dependency:** `@ai-sdk/openai`
- **New UI components:** shadcn dropdown-menu, textarea
- **New file:** `frontend/lib/ai/models.ts` — model registry
- **Modified:** `frontend/lib/ai/agent.ts` — multi-provider model selection
- **Modified:** `frontend/app/api/chat/stream/route.ts` — accepts `model` param in FormData
- **Modified:** `frontend/stores/chat-store.ts` — `selectedModel` state, passed in sendMessage
- **Modified:** `frontend/components/chat/MessageInput.tsx` — model selector with animated transitions and provider icons
- **Env:** Added `OPENAI_API_KEY` to `.env`

## Out of Scope

- Per-user API key management (BYOK)
- Model-specific system prompts
- Cost/usage tracking per model
- Persisting model preference across sessions

## Done Criteria

- [x] User can select a model from dropdown before sending a message
- [x] Selected model is sent to API and used for response generation
- [x] Both Gemini and OpenAI models produce streaming responses
- [x] TypeScript compiles without errors
