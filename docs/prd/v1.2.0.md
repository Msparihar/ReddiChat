# PRD: ReddiChat v1.2.0 — Production Hardening

**Status:** Draft
**Date:** 2026-03-28

## Version Goal

Make ReddiChat production-safe by adding rate limiting, input validation, database optimizations, connection pooling, and proper error handling — so it can handle real users without crashing or being abused.

## Features

### 1. Database Connection Pooling & Timeouts
- Configure pg pool with max connections (20), idle timeout (30s), connection timeout (2s), query timeout (30s)
- Add pool error handler to prevent silent crashes
- **File:** `frontend/lib/db/index.ts`

### 2. Database Indexes
- Add indexes for high-frequency queries:
  - `conversations(user_id, updated_at DESC)` — conversation list
  - `messages(conversation_id)` — message history
  - `messages(user_id)` — user message lookup
  - `file_attachments(user_id)` — attachment cleanup
  - `message_attachments(file_attachment_id)` — join optimization
- Fix N+1 query in `/api/chat/history/[id]/route.ts` — use JOIN instead of Promise.all loop
- **Files:** `frontend/lib/db/schema.ts`, `frontend/app/api/chat/history/[id]/route.ts`

### 3. Rate Limiting
- Add per-user rate limits on critical endpoints:
  - `/api/chat/stream` — 30 requests/hour
  - `/api/upload` — 20 uploads/hour
  - `/api/reddit/user/[username]/*` — 60 requests/hour
- Use in-memory rate limiter (no Redis dependency for now) or Upstash if available
- Return 429 with clear error message and retry-after header
- **Files:** New `frontend/lib/rate-limit.ts`, all API routes

### 4. Input Validation (Zod)
- Validate all API inputs with Zod schemas:
  - Chat message: min 1 char, max 5000 chars
  - Conversation ID: valid UUID format
  - Model ID: must be from allowed model list
  - Reddit username: alphanumeric + underscore/hyphen, 3-20 chars
  - File uploads: validate MIME type matches extension, enforce 10MB limit server-side
- **Files:** New `frontend/lib/validation.ts`, all API routes

### 5. External API Error Handling
- Reddit token refresh: add retry with exponential backoff (3 attempts)
- S3 uploads: add 10s timeout wrapper
- AI streaming: only save assistant message to DB after full stream completes (prevent partial messages)
- Structured error responses: categorize errors (rate_limit, auth, network, validation, internal)
- **Files:** `frontend/lib/reddit/index.ts`, `frontend/lib/s3/index.ts`, `frontend/app/api/chat/stream/route.ts`

### 6. Better Auth Secret
- Replace hardcoded `my_super_secret_key_for_better_auth` with a proper 32+ char random secret
- Document in `.env.local.example`
- **File:** `frontend/.env`, `frontend/.env.local.example`

### 7. Structured Logging
- Replace all `console.log`/`console.error` in API routes with structured logger
- Include request context: userId, endpoint, duration, status
- Log external API failures with retry count and error details
- **Files:** New `frontend/lib/logger.ts`, all API routes
